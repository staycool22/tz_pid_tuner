# pass_through PID Test Skill

## 1. 适用范围

本 skill 只用于 `CONTROL_MODE_PASS_THROUGH` 调参，不直接套用于普通 `CONTROL_MODE_POS` 或 `CONTROL_MODE_SPEED`。

这个接口的真实结构不是单一 PID，而是一个级联控制:

```text
pos      -> 位置环 -> 速度修正量
rpm      -> 速度基准
current  -> 电流前馈

speed_cmd = rpm + pos_loop_output
iq_cmd    = current + speed_loop_output
```

因此调参时必须分开看 3 件事:

- `pos` 决定位置环给出的速度修正。
- `rpm` 决定速度环的主目标。
- `current` 会直接叠加到最终 `iq`，会绕过 PID 的一部分调节效果。

## 2. 大模型必须遵守的硬规则

### 2.1 先稳内环，再动外环

固定顺序:

1. 先确认接口输入方向、零位、编码器方向正确。
2. 先把 `current = 0`，只调速度环。
3. 速度环稳定后，再调位置环。
4. 最后才加 `current` 前馈做负载补偿。

禁止一开始同时改位置环、速度环、电流前馈。

### 2.2 默认起步策略

默认推荐:

- 速度环先用 `PI`。
- 位置环先用纯 `P`。
- `p_pid_kd_proc` 只在位置环已经基本稳定后，才允许小幅加入。
- `p_pid_ki` 默认最后考虑，且通常很小或先保持 0。

原因:

- `pass_through` 的位置环输出是速度修正量，不是电流。
- 外环一旦 D/I 给大，很容易把内环扰乱成速度往返摆动。
- 位置保持工况下，很多微小抖动并不是位置环 P 太小，而是内层速度环或负载补偿不足。

### 2.3 先排机制，再改 PID

出现低速抖动、停不稳、回位慢时，优先检查这些非 PID 机制:

1. `s_pid_min_erpm` 是否让速度环低速退出。
2. `current` 是否未清零，正在直接推电机。
3. `s_pid_ramp_erpms_s` 是否把速度目标爬坡得太慢。
4. `s_pid_allow_braking` 是否关闭，导致只能推不能反拖。
5. `l_current_max`、`l_current_max_scale`、`lo_current_max` 是否让可用电流过小。
6. `p_pid_ang_div` 是否改变了位置误差尺度。

### 2.4 验收优先级

判断参数是否更好时，优先级按下面顺序:

1. 位置是否稳定保持，不出现肉眼可见来回摆动。
2. 带载时是否能可靠站住，不缓慢滑落。
3. 小步进命令后是否能回到目标附近，不持续打摆。
4. 速度波动是否可接受。
5. 单一测试点的 RPM 曲线是否足够漂亮。

不要为了追求某一个速度点的平滑曲线，破坏位置保持稳定性。

## 3. 可观测边界与推理规则

如果大模型只能看到运行日志或上位机状态量，而看不到全部内部变量，则必须区分三类信息:

### 3.1 可直接依据的现象

- 实际转速 `rpm`
- 实际电流 `current`
- 实际位置 `pid_pos_now`
- 目标位置变化后的响应速度、超调、回摆、抖动

### 3.2 只能作为实现假设的机制

以下内容通常不能仅凭运行结果直接证明，需要结合配置或源码确认:

- `s_pid_min_erpm` 是否非 0
- `s_pid_allow_braking` 是否关闭
- `s_pid_ramp_erpms_s` 是否生效
- `l_current_max_scale` 是否导致限流变紧
- `p_pid_gain_dec_angle` 是否在小误差区衰减了增益

### 3.3 输出结论时的表达规范

当证据不足时，必须写成:

- 可能由 `s_pid_min_erpm` 导致低速速度环退出，需人工核查配置
- 可能是 `current` 前馈仍在生效，需确认命令下发值
- 可能受限流或制动开关影响，需检查配置项

禁止把未观测到的内部配置当成既定事实。

## 4. 真正可执行的调参 Skill

### 4.1 第一步: 接口连通性与方向确认

先确认:

- `pos` 增大时，位置反馈方向正确。
- `rpm` 为正时，电机方向与预期一致。
- `current` 为正时，输出力矩方向正确。
- 编码器零位、方向、`p_pid_offset` 没有明显错误。

如果方向都不一致，先修接口和传感器配置，不进入 PID 调参。

### 4.2 第二步: 速度环单独建参

执行方式:

- 固定 `pos`，尽量让位置误差接近 0。
- 设置 `current = 0`。
- 给定中低速 `rpm` 阶跃，先看速度跟随。

推荐顺序:

1. 先调 `s_pid_kp`，调到有足够跟随，但不明显振荡。
2. 再决定是否需要 `s_pid_kd`，只用来压超调或减小速度回摆。
3. 最后加 `s_pid_ki`，用于消除负载下的持续速度偏差。

判断逻辑:

- 跟随太慢: 优先加 `s_pid_kp`。
- 有持续偏差、负载下顶不住: 再加 `s_pid_ki`。
- 过冲明显、反复拉扯: 先减 `s_pid_kp`，再少量试 `s_pid_kd`。
- 低速噪声大: 先减小 `s_pid_kd` 或增大 `s_pid_kd_filter`。

### 4.3 第三步: 位置环单独建参

前提:

- 速度环已经在中低速下稳定。
- `current` 仍然保持 0。

推荐顺序:

1. 先调 `p_pid_kp`，让位置误差能被有效转成速度修正。
2. 如果接近目标点有轻微回摆，可少量加 `p_pid_kd_proc`。
3. `p_pid_kd` 只在误差变化项确实需要抑制时少量尝试。
4. `p_pid_ki` 最后考虑，且一般很谨慎。

判断逻辑:

- 推不动、回位慢: 先加 `p_pid_kp`。
- 靠近目标点打摆: 先减 `p_pid_kp`，或少量加 `p_pid_kd_proc`。
- 小误差区发软: 检查 `p_pid_gain_dec_angle` 是否过大。
- 长期静差明显: 再考虑少量 `p_pid_ki`，但优先确认速度环与前馈是否不足。

### 4.4 第四步: 加入电流前馈

使用目的:

- 抵消重力或已知负载。
- 降低速度环积分压力。
- 提升起步和保持时的力矩余量。

加前馈的规则:

- 只在速度环和位置环都基本稳定后加入。
- 从小值开始增加。
- 每次增加后都要复查位置保持抖动是否变小、是否更容易过推。

如果加前馈后速度 PID 看起来没作用，先怀疑前馈掩盖了速度环，而不是直接怀疑 PID 全失效。

## 5. 特殊问题处理 Skill

### 5.1 位置环纯 P + 速度环 PI 是否可行

这是 `pass_through` 的默认优先方案，而且通常是最稳妥的起点。

原因:

- 位置环输出的是速度修正量，纯 P 已经能形成偏差越大，速度修正越强的外环行为。
- 速度环 PI 更适合补偿负载和摩擦，尤其是静态保持时需要持续电流。
- 位置环过早加入 I/D，容易把小误差放大成持续的速度抖动。

结论:

- 默认先用位置环纯 P + 速度环 PI。
- 只有在速度环已经很稳、位置端仍有轻微回摆时，才少量尝试 `p_pid_kd_proc`。

### 5.2 低速退出逻辑是什么意思

源码逻辑是:

```text
if abs(m_speed_pid_set_rpm) < s_pid_min_erpm:
    速度环积分清零
    速度环返回 0
```

这意味着:

- 当目标速度过低时，速度环不再参与输出。
- 但 `pass_through` 外层仍然会把 `current` 前馈直接叠加到最终 `iq`。

所以低速时可能出现两种误判:

- 你以为速度 PID 失效了，其实是被阈值关掉了。
- 你以为 0 速时不该有出力，但实际上是 `current` 还在推动电机。

### 5.3 `Minimum ERPM = 0` 是否可以运作

可以运作，代码路径上没有问题，但风险是:

- 速度环会一直尝试参与极低速和接近静止区控制。
- 此时速度估计噪声、编码器量化、齿槽和摩擦都会被放大。
- 低速抖动和来回找零现象可能更明显。

建议:

- 如果当前低速已经明显抖动，不要把 `Minimum ERPM` 设为 0 作为第一解法。
- 更稳妥的方式是先确认前馈、速度环 I、位置环 P 是否匹配，再决定要不要降低门限。

### 5.4 设置门限能否改善低速抖动

可能可以，但本质是放弃一部分低速速度闭环参与度来换取稳定。

规律是:

- 门限调大: 更容易减轻低速抖动，但低速响应会变钝。
- 门限调小: 低速响应更积极，但更容易把速度估计噪声放大成抖动。

因此它不是单纯越大越好或越小越好，而是稳定性和低速响应的折中。

### 5.5 为什么某些机械臂速度环 I 很大，反而更稳

常见原因不是系统更先进，而是负载更需要保持电流。

在机械臂场景下，速度环 I 项可能会积出一个稳定的保持电流，用来抵消:

- 重力矩
- 减速器摩擦差
- 机构偏载
- 静摩擦门槛

于是现象上会看到:

- 速度环 RPM 不是特别漂亮。
- 但位置却更稳，不再上下抖动 `0.3`。

这说明工程目标应优先看能否稳定站住，而不是只看速度曲线是否最平。

### 5.6 为什么空载和带载 PID 差很多

这是正常现象，因为负载会直接改变系统动力学:

- 惯量变化
- 重力矩变化
- 摩擦变化
- 反向间隙和减速器弹性影响变明显

在 `pass_through` 这种级联结构里，差异会被放大:

- 外环位置误差会变成速度修正。
- 内环速度误差再变成电流。
- 任何负载变化都会同时改变两层环的感受。

实践规则:

- 如果实际应用以带载为主，就优先按带载工况建参。
- 空载只作为补充验证，不要用空载最优参数强行覆盖带载需求。

### 5.7 调试顺序应该是什么

推荐固定顺序:

1. 接口方向、零位、编码器正确。
2. 前馈 `current = 0`。
3. 先调速度环。
4. 再调位置环。
5. 复查低速行为和停稳行为。
6. 最后再加电流前馈。
7. 用代表工况做回归验证。

### 5.8 速度环应先调低速还是中高速

推荐顺序是:

1. 先用中低速建参。
2. 再去低速修边。
3. 再用中高速验证边界。
4. 最后回到低速复核停稳质量。

原因:

- 纯低速区最容易受速度观测噪声和摩擦影响，不适合作为初始建参点。
- 中低速更容易看出 `kp/ki/kd` 的主趋势。
- 参数定型后，再处理低速特殊机制更高效。

### 5.9 `减速比 48 + 极对数 11 + 1000 ERPM` 属于什么速度区

按输出轴机械转速估算:

```text
电机机械 rpm = 1000 / 11  90.9 rpm
输出轴 rpm   = 90.9 / 48  1.89 rpm
```

这个点更接近低速验证点而不是典型中速点。

如果在 `1000 ERPM` 下上下跳动 `100 ERPM`:

- 从电角速度看，相当于约 10% 波动，不算漂亮。
- 但如果机械臂位置能稳定站住、没有明显 `0.3` 上下扰动，工程上可能仍然可接受。

判断时要优先看末端位置稳定性，而不是只盯着 ERPM 数字。

### 5.10 速度有波动但位置能稳住，该怎么取舍

优先保位置稳定。

工程上更实用的判断是:

- 如果位置保持稳定、负载下不滑落、指令响应可接受，那么允许存在一定速度估计波动。
- 只有当速度波动已经反映成可见抖动、噪音、发热或冲击时，才需要继续压速度环波动。

结论:

- 不要为了把 RPM 曲线修得更漂亮，牺牲位置稳定性。

### 5.11 大模型在有限工况下如何做泛化

不可能覆盖所有工况，因此必须采用代表工况 + 保守泛化的策略。

大模型应遵守:

1. 优先用最接近真实应用的带载工况做主判断。
2. 至少覆盖起步、小步进、保持、反向、较高速度几类代表动作。
3. 任何参数修改后，都检查是否恶化了位置保持这一最高优先级指标。
4. 当不同工况冲突时，优先保住主工况和位置稳定，不追求所有工况都绝对最优。
5. 对未覆盖工况，只能给出保守预测，不能宣称已经最优。

## 6. 大模型输出建议格式

每次给调参建议时，建议固定输出 5 段:

1. `当前现象`
2. `最可能原因`
3. `需人工核查的配置项`
4. `建议修改顺序`
5. `修改后的验收标准`

建议模板:

```text
当前现象:
- 例如: 低速保持时有约 0.3 微抖动，但位置总体能站住。

最可能原因:
- 速度环 I 不足，无法积出稳定保持电流。
- 或低速区速度估计噪声较大，导致速度环持续微调。

需人工核查的配置项:
- s_pid_min_erpm
- current 前馈
- s_pid_allow_braking
- l_current_max_scale

建议修改顺序:
1. 保持 current=0，先小幅增加 s_pid_ki。
2. 若低速噪声明显，再减小 s_pid_kd 或增大 s_pid_kd_filter。
3. 若仍是接近目标点回摆，再小幅减 p_pid_kp 或少量加 p_pid_kd_proc。

修改后的验收标准:
- 位置保持更稳。
- 不出现明显来回摆动。
- 不引入更大的超调或发热。
```

## 7. 一句话版本

`pass_through` 调参的核心 skill 只有一句话:

```text
先把 current 前馈拿掉，先稳速度环，再稳位置环，最后再用前馈补负载，并始终优先保证位置稳定而不是追求某个速度点曲线最好看。
```
