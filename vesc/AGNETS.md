# AGNETS.md

## 1. 文件目的

本文件定义本项目中“单一自动调参程序 + 人工操作员”在构建 `pass_through` PID 自整定系统时必须遵守的统一规则。

当前项目采用单一自动调参主程序，所有实验执行、参数搜索、结果判断与报告输出，均由该程序完成。

目标不是让程序自由试错，而是让全部实现都严格服从 `pass_through` 的真实控制结构与安全边界。

## 2. 统一背景

### 2.1 控制结构

本项目控制对象为 `CONTROL_MODE_PASS_THROUGH`，其本质是级联控制，而不是 MIT 位置速度直接映射电流模式：

```text
pos      -> 位置环 -> 速度修正量
rpm      -> 速度基准
current  -> 电流前馈

speed_cmd = rpm + pos_loop_output
iq_cmd    = current + speed_loop_output
```

### 2.2 系统必须记住

1. 位置环调的是“速度修正能力”，不是直接调力矩。
2. 速度环调的是“电流修正能力”，不是直接调电压。
3. `current` 是绕过部分 PID 评估路径的直接前馈量。
4. 任何不区分 `pos / rpm / current` 作用来源的分析，都视为不合格分析。

## 3. 角色定义

### 3.1 人工操作员

职责：

1. 确认机械装置、线缆、编码器、限位与急停可用。
2. 提供真实应用工况和优先级。
3. 在高风险实验前做人身和设备安全确认。
4. 对异常振动、异响、发热进行人工兜底判断。

### 3.2 自动调参主程序

职责：

1. 按固定流程执行实验。
2. 自动记录全部参数和数据。
3. 自动评分、比较与回退。
4. 在超阈值或异常时立即停止当前试验。
5. 在证据不足时输出“需人工核查”的保守结论。
6. 在任何阶段都不得越过安全壳直接做激进搜索。

## 4. 硬规则

以下规则必须被自动调参主程序严格遵守：

1. 不得在初始阶段同时修改位置环、速度环与电流前馈。
2. 不得跳过方向、零位、编码器检查直接开始自动整定。
3. 不得在 `current != 0` 时把观察结果当作纯 PID 效果。
4. 不得仅凭运行现象断言 `s_pid_min_erpm`、`s_pid_allow_braking` 等内部配置已经确定。
5. 不得为了追求某个速度点曲线平滑，牺牲位置保持稳定性。
6. 不得在没有回退机制的前提下做大步参数搜索。
7. 不得把单次幸运结果当作稳定收敛结论，必须经过重复验证。

## 5. 固定调参顺序

自动调参主程序必须按照以下顺序工作：

1. 检查方向、零位、编码器方向、`p_pid_offset`
2. 抓取当前参数快照
3. 强制 `current = 0`
4. 单独调速度环
5. 单独调位置环
6. 复查低速和保持行为
7. 最后才允许加入 `current` 前馈
8. 用代表工况做回归验证

## 6. 默认参数策略

如果没有额外证据，默认采用：

1. 速度环：先 `PI`
2. 位置环：先 `P`
4. `p_pid_kd`：谨慎使用

理由：

1. 位置环输出的是速度修正量，过早加入外环 I/D 容易搅乱内环。
2. 速度环更适合负责负载与保持电流补偿。

## 7. 可观测与不可观测边界

### 7.1 可直接依据的现象

1. 实际转速
2. 实际电流
3. 实际位置
4. 响应时间
5. 超调
6. 回摆
7. 抖动
8. 保持时是否滑落

### 7.2 只能作为候选原因的机制

以下项目若没有配置快照或源码确认，只能写成“可能”：

1. `s_pid_min_erpm` 触发低速退出
2. `s_pid_ramp_erpms_s` 导致目标变软
3. `s_pid_allow_braking` 导致不能主动反拖
4. `l_current_max_scale` 导致限流收紧
5. `p_pid_gain_dec_angle` 导致小误差区增益被削弱
6. `current` 前馈掩盖 PID 效果

### 7.3 结论表达模板

当证据不足时，必须使用这样的措辞：

1. 可能由 `s_pid_min_erpm` 导致低速速度环退出，需人工核查配置。
2. 可能是 `current` 前馈仍在生效，需确认命令下发值。
3. 可能受制动开关或限流影响，需检查配置项。

## 8. 实验策略

### 8.1 速度环实验

约束：

1. 固定 `pos`
2. `current = 0`
3. 优先使用中低速阶跃

推荐调参顺序：

1. 先调 `s_pid_kp`
2. 再决定是否需要 `s_pid_kd`
3. 最后用 `s_pid_ki` 消除持续偏差

### 8.2 位置环实验

前提：

1. 速度环已稳定
2. `current = 0`

推荐调参顺序：

1. 先调 `p_pid_kp`
2. 接近目标点回摆时再试 `p_pid_kd_proc`
3. 再评估是否真的需要 `p_pid_kd`
4. 最后再看 `p_pid_ki`

### 8.3 前馈实验

前提：

1. 速度环和位置环都基本可用
2. 存在明确负载补偿需求

执行要求：

1. 从很小的 `current` 起步
2. 每次修改后复查位置保持和过推风险
3. 不能把前馈改善误判成 PID 已最优

## 9. 自动调参算法约束

第一版允许的策略：

1. 规则驱动搜索
2. 小步扫描
3. 局部网格搜索
4. 带回退的启发式迭代

第一版不建议直接采用：

1. 无约束随机大范围搜索
2. 无安全保护的强化学习
3. 直接把黑盒优化器连接到真机高风险参数空间

## 10. 评价优先级

所有自动评分和程序生成的建议都必须遵守以下优先级：

1. 位置是否稳定保持
2. 带载时是否能可靠站住
3. 小步进后是否能回到目标附近
4. 是否存在明显往返摆动
5. 速度波动是否可接受
6. 某单一速度点曲线是否漂亮

如果发生冲突，必须优先保住前 1 到 4 项。

## 11. 必查配置项

当出现低速抖动、停不稳、回位慢、反向不干净时，自动调参主程序必须优先检查并在报告里列出：

1. `s_pid_min_erpm`
2. `current` 前馈是否为零
3. `s_pid_ramp_erpms_s`
4. `s_pid_allow_braking`
5. `l_current_max`
6. `l_current_max_scale`
7. `lo_current_max`
8. `p_pid_ang_div`
9. `p_pid_gain_dec_angle`

## 12. 安全机制要求

程序实现必须具备：

1. 参数写入前快照备份
2. 当前实验失败时的自动回退
3. 峰值电流阈值保护
4. 位置偏差越界保护
5. 采样中断或通讯异常保护
6. 人工急停优先权
7. 每次实验的最大持续时间限制

## 13. 报告输出格式

后续所有自动报告和程序生成的建议，默认按下面 5 段输出：

1. 当前现象
2. 最可能原因
3. 需人工核查的配置项
4. 建议修改顺序
5. 修改后的验收标准

建议模板：

```text
当前现象:
- 低速保持时有轻微抖动，但位置总体能站住。

最可能原因:
- 速度环 I 不足，无法积出稳定保持电流。
- 或低速区速度估计噪声较大，导致速度环持续微调。

需人工核查的配置项:
- s_pid_min_erpm
- current 前馈
- s_pid_allow_braking
- l_current_max_scale

建议修改顺序:
1. 保持 current=0，先小幅增加 s_pid_ki。
2. 若低速噪声明显，再减小 s_pid_kd 或增大 s_pid_kd_filter。
3. 若仍是接近目标点回摆，再小幅减 p_pid_kp 或少量加 p_pid_kd_proc。

修改后的验收标准:
- 位置保持更稳。
- 不出现明显来回摆动。
- 不引入更大的超调或发热。
```

## 14. 推荐的程序实现原则

程序实现应优先：

1. 先搭安全壳，再写搜索器。
2. 先做日志和回退，再做参数推荐。
3. 先支持规则驱动，再扩展高级优化。
4. 所有关键结论都可追溯到原始测试数据。

不推荐：

1. 一开始就把程序写成难以解释的大一体化搜索器。
2. 用隐式状态机隐藏关键实验步骤。
3. 把安全判断散落在多个难以追踪的模块里。

## 15. 本项目的一句话原则

围绕 `pass_through` 做 PID 自整定时，单一自动调参程序必须遵守同一条主线：

先去掉 `current` 前馈，先稳速度环，再稳位置环，最后再用前馈补负载，并始终把位置稳定性放在速度曲线美观之前。
