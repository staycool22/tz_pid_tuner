# pass_through PID 自整定构筑计划

## 1. 项目目标

构建一套面向 `CONTROL_MODE_PASS_THROUGH` 的 PID 自整定方案，目标不是直接追求某一条漂亮的转速曲线，而是围绕真实控制结构完成以下能力：

当前版本默认采用“共享底层能力 + 两个独立整定程序 + 人工监护”的工作模式，不再把速度环与位置环耦合到同一个调参程序中。

1. 安全地下发 `pass_through(pos, rpm, current)` 测试指令。
2. 自动采集位置、速度、电流等关键观测量。
3. 通过独立的速度环整定程序和位置环整定程序分别完成调试。
4. 依据位置稳定性、带载保持能力、响应质量与电流使用情况进行评分。
5. 输出可复现的参数建议、实验记录和验收结论。

## 2. 控制对象理解

### 2.1 控制本质

该接口不是 MIT 模式，也不是单环位置 PID，而是三输入级联控制：

```text
pos      -> 位置环 -> 速度修正量
rpm      -> 速度基准
current  -> 电流前馈

speed_cmd = rpm + pos_loop_output
iq_cmd    = current + speed_loop_output
```

因此自整定程序必须遵守：

1. 位置环输出单位本质上是速度修正量。
2. 速度环输出单位本质上是电流修正量。
3. `current` 会直接叠加到最终 `iq`，不能与 PID 效果混在一起评估。
4. 任何整定结论都必须说明当前 `current` 是否为 `0`。

### 2.2 由图片得到的补充理解

图片强调的是“三环前馈透传”而非 MIT 直控模式，因此程序设计应偏向：

1. 把位置精度和带载稳定性作为最高目标。
2. 将动力学补偿视为后置增强项，而不是第一阶段主控制手段。
3. 允许未来扩展“速度前馈 / 力矩前馈 / 重力补偿”，但第一版不应依赖这些高级补偿才能工作。

## 3. 必须固化的调参原则

程序设计中必须把下列规则写成硬约束，而不是仅作为注释：

1. 先确认方向、零位、编码器配置，再进入任何整定程序。
2. 速度环整定程序运行时强制 `current = 0`，且不允许位置环参与搜索。
3. 位置环整定程序运行时默认 `current = 0`，且只允许在速度环参数已冻结的前提下工作。
4. 电流前馈补偿必须在速度环与位置环都完成后，再由独立步骤处理。
5. 默认起步策略：
   - 速度环只使用 `PI`
   - 位置环先 `P`
   - `p_pid_kd_proc` 只做小量阻尼修饰
   - `p_pid_ki` 默认最后再考虑
   - 速度环不引入 `D` 项
6. 若证据不足，不得把 `s_pid_min_erpm`、`s_pid_allow_braking`、`l_current_max_scale` 等内部配置状态当作既定事实。

## 4. 第一版范围

### 4.1 包含内容

1. 单轴电机 `pass_through` 的速度环独立整定程序。
2. 单轴电机 `pass_through` 的位置环独立整定程序。
3. 中低速阶跃实验。
4. 小步进位置实验。
5. 保持实验。
6. 反向实验。
7. 带日志与参数快照的结果输出。

### 4.2 暂不包含

1. 多轴联动整定。
2. 全自动动力学建模。
3. 复杂轨迹跟踪最优控制。
4. 依赖高精度外部力传感器的力控闭环。
5. 无人工监护下的大电流极限搜索。

## 5. 系统架构建议

建议采用“共享基础模块 + 独立整定程序”的结构，而不是一个大一体化调参器：

```text
autotune/
  common/
    config/         设备与试验配置
    io/             通讯与指令下发
    acquisition/    数据采集与时间戳对齐
    analysis/       指标提取、评分、异常检测
    reporting/      结果报告、参数建议、实验归档
    safety/         急停、越界、回退、阈值保护
  speed_tuner/
    scenarios/      固定速度设定点、反向、稳态测试
    tuning/         速度环参数搜索
    app/            速度环整定程序入口
  position_tuner/
    scenarios/      小步进、保持、换向到位测试
    tuning/         位置环参数搜索
    app/            位置环整定程序入口
  feedforward_tool/
    app/            电流前馈补偿扫描工具
```

### 5.1 `common/config`

职责：

1. 保存电机基本信息、极对数、减速比、编码器方向。
2. 保存限流、限速、单次动作幅度、回退阈值。
3. 保存要读写的 VESC 参数映射关系。

### 5.2 `common/io`

职责：

1. 封装 `pass_through(pos, rpm, current)` 下发。
2. 读取实际速度、实际电流、位置反馈、故障状态。
3. 写入和回读 PID 参数。
4. 提供紧急停机和恢复默认参数接口。

### 5.3 `common/acquisition`

职责：

1. 统一时间戳。
2. 缓存原始样本。
3. 记录测试场景标签、参数版本、配置快照。
4. 为分析模块输出结构化时序数据。

### 5.4 `speed_tuner/scenarios`

速度环整定程序建议实现 3 类场景：

1. 固定速度设定点测试：
   - 下发固定 `rpm`
   - 记录速度读回曲线用于整定 `PI`
2. 稳态速度保持：
   - 检查稳态误差与电流占用
3. 反向速度切换：
   - 检查制动、超调和换向稳定性

### 5.5 `position_tuner/scenarios`

位置环整定程序建议实现 3 类场景：

1. 小步进位置测试：
   - 固定较小角度步进
   - `current = 0`
   - 使用已冻结的速度环参数
2. 保持测试：
   - 观察目标点附近是否有抖动、滑落、持续误差
3. 反向到位测试：
   - 检查位置回位、换向回摆和到位稳定性

### 5.6 `common/analysis`

核心指标建议：

1. 速度响应时间
2. 速度超调
3. 速度稳态误差
4. 位置到位时间
5. 位置超调
6. 目标附近抖动幅值
7. 保持阶段平均电流
8. 峰值电流
9. 反向冲击程度
10. 是否触发保护或异常退出

分析模块应支持按“速度环专用指标”和“位置环专用指标”分别出报告，避免混合打分。

### 5.7 `speed_tuner/tuning` 与 `position_tuner/tuning`

建议采用“保守、独立、可回退”的搜索方式：

1. 第一版优先使用规则驱动 + 小步迭代。
2. 速度环与位置环分别维护自己的搜索空间和评分函数。
3. 速度环采用固定速度设定点测试，并借鉴 Ziegler-Nichols 整定思想生成 `PI` 初值。
4. 每次只改一小组参数。
5. 每次实验完成后只与本环上一版参数打分比较。
6. 如果更差或触发异常，自动回退到本环上一稳定版本。
7. 不建议一开始直接上黑盒全局优化。

### 5.8 `common/reporting`

输出内容至少包括：

1. 本轮测试的设备信息。
2. 当前配置快照。
3. 速度环程序尝试过的参数。
4. 位置环程序尝试过的参数。
5. 各场景指标表。
6. 最优参数与次优参数。
7. 未确认的风险项与需人工核查的配置项。

## 6. 独立程序流程设计

### 6.1 通用上电前检查

1. 核对方向：
   - `pos` 正向
   - `rpm` 正向
   - `current` 正向
2. 核对编码器零位和 `p_pid_offset`。
3. 拉取当前 VESC 参数快照。
4. 检查限流、限速、安全阈值是否在允许范围内。

### 6.2 速度环整定程序

程序边界：

1. 固定 `pos`，尽量让位置误差接近 0。
2. 强制 `current = 0`。
3. 使用固定速度设定点进行测试，可覆盖多个中低速目标值。
4. 不修改位置环参数。
5. 速度环只整定 `s_pid_kp` 与 `s_pid_ki`，不整定 `s_pid_kd`。

整定思想：

1. 参考 Ziegler-Nichols 方法的思路，用固定速度设定点观察闭环响应。
2. 先逐步增加 `s_pid_kp`，寻找接近临界振荡但尚可控的增益区间。
3. 结合回读速度波形估计临界振荡周期或等效响应特征。
4. 由临界增益和响应周期换算一组 `PI` 初值。
5. 再通过实测稳态误差、超调和收敛时间对 `kp / ki` 做保守微调。

调参顺序：

1. 先扫 `s_pid_kp`。
2. 在接近临界振荡时记录读回速度曲线特征。
3. 再加入 `s_pid_ki` 消除持续偏差。
4. 通过固定速度回读数据微调 `s_pid_kp / s_pid_ki`。

停止条件：

1. 速度跟随足够快。
2. 速度超调可接受。
3. 不出现明显持续振荡。
4. 峰值电流未越界。

输出产物：

1. 冻结版速度环参数集。
2. 速度环实验报告。
3. 供位置环整定程序导入的速度环基线配置。

### 6.3 位置环整定程序

程序前提：

1. 速度环整定程序已通过。
2. 仍保持 `current = 0`。
3. 只读取并使用冻结的速度环参数，不回写速度环搜索结果。

调参顺序：

1. 先扫 `p_pid_kp`。
2. 接近目标点若回摆，再少量加 `p_pid_kd_proc`。
3. `p_pid_kd` 只在确有必要时尝试。
4. `p_pid_ki` 最后再考虑。

停止条件：

1. 小步进后能稳定回位。
2. 目标附近无肉眼可见往返摆动。
3. 带轻载时不明显滑落。

输出产物：

1. 冻结版位置环参数集。
2. 位置环实验报告。
3. 与速度环基线组合后的推荐参数包。

### 6.4 前馈补偿工具

前提：

1. 速度环和位置环都已基本稳定。
2. 有明确负载补偿需求。

执行方式：

1. 从小 `current` 前馈起步。
2. 观察保持电流、位置抖动、过推风险。
3. 与无前馈版本对比是否真正改善。

判据：

1. 位置保持更稳。
2. 速度环积分压力下降。
3. 不引入新的振荡或明显发热。

## 7. 评分函数建议

第一版应分别为速度环程序和位置环程序建立评分函数，方向是“先排除不安全，再比较性能”。

### 7.1 硬性淘汰项

满足任一项直接判失败：

1. 峰值电流超安全阈值。
2. 出现明显持续振荡。
3. 丢反馈或通讯异常。
4. 位置偏离超安全窗口。
5. 电机出现异常保护或人工紧停。

### 7.2 速度环软评分项

建议评分权重：

1. 速度稳态误差：30%
2. 速度超调与响应时间：30%
3. 临界振荡附近的可辨识性与收敛质量：20%
4. 电流使用效率与温和性：20%

### 7.3 位置环软评分项

建议评分权重：

1. 位置保持稳定性：35%
2. 带载不滑落能力：20%
3. 小步进响应质量：25%
4. 回摆与到位稳定性：10%
5. 电流使用效率与温和性：10%

说明：

1. 速度环程序不以位置指标作为主评分。
2. 速度环程序默认采用 `PI`，不使用 `D` 项，以降低低速噪声对整定结果的污染。
3. 位置环程序建立在已稳定的速度环参数之上。
4. 若位置稳定和速度曲线冲突，优先选择位置更稳的参数。

## 8. 需要重点识别的干扰因素

两个整定程序都必须显式检查或提示这些机制：

1. `s_pid_min_erpm`
   - 可能导致低速速度环直接退出
2. `s_pid_ramp_erpms_s`
   - 可能让目标响应看起来偏软
3. `s_pid_allow_braking`
   - 可能导致只能推不能反拖
4. `l_current_max` / `l_current_max_scale` / `lo_current_max`
   - 可能导致实际可用电流不足
5. `p_pid_ang_div`
   - 会直接改变位置误差尺度
6. `p_pid_gain_dec_angle`
   - 可能让目标附近表现发软
7. `current` 非零
   - 可能掩盖速度环与位置环真实效果

## 10. 最小可用版本里程碑

### M1: 通讯与安全闭环

交付：

1. 可读写参数
2. 可发 `pass_through`
3. 可采集基础数据
4. 可紧急停机

### M2: 速度环独立整定程序

交付：

1. 自动执行固定速度设定点测试
2. 自动计算响应指标
3. 基于 Ziegler-Nichols 思路自动推荐 `s_pid_kp / ki`
4. 生成冻结版速度环参数

### M3: 位置环独立整定程序

交付：

1. 自动执行小步进与保持测试
2. 自动推荐 `p_pid_kp / kd_proc / kd / ki`
3. 读取冻结版速度环参数后独立工作

### M4: 前馈与报告

交付：

1. 电流前馈补偿扫描
2. 速度环与位置环分开报告
3. 参数对比与回退记录

## 11. 建议的首版实现策略

首版不追求“完全无人值守最优”，而应追求：

1. 不危险
2. 可解释
3. 可复现
4. 能逐阶段稳定收敛

因此建议：

1. 先做速度环规则驱动整定程序。
2. 再做位置环规则驱动整定程序。
3. 把每一步为何调整写进各自报告。
4. 保留人工确认点。
5. 待样本足够后，再考虑加入贝叶斯优化或学习型搜索。

## 12. 验收标准

第一版程序可认为达到可用状态，当且仅当：

1. 能分别形成可独立运行的速度环整定程序和位置环整定程序。
2. 位置环程序只基于冻结的速度环参数工作，不与速度环搜索逻辑耦合。
3. 能明确区分速度环、位置环与前馈三部分贡献。
4. 遇到异常时能安全停机并回退参数。
5. 输出结果能让人工明确回答：
   - 当前现象是什么
   - 最可能原因是什么
   - 哪些配置仍需人工核查
   - 建议下一步如何修改
   - 修改后如何验收

## 13. 一句话策略

这套自整定方案的核心不是“把两层 PID 耦合进一个搜索器里暴力搜索”，而是把 `pass_through` 的级联结构拆开，用两个独立程序先稳住速度环、再稳住位置环，最后才用电流前馈补负载，并始终把位置稳定性放在最高优先级。
